# v0.2 Contract Boundary

## Goal

v0.2 expands the AI-operable 2D contract without turning GIS Engine into a
renderer rewrite. The release should make generated maps easier to validate,
replay, diagnose, and export.

## In Scope

| Area | Contract Work | Exit Criteria |
| --- | --- | --- |
| MCP tool hardening | all tool inputs validated, failures return `Diagnostic[]` | `pnpm test:ai` covers invalid input paths |
| Command schema strictness | command schemas reject unknown public fields | schema-sync test rejects unknown command fields |
| Expression v0.2 | support `case`, `match`, `zoom`, `to-number`, `to-string` | validator tests and adapter capability list align |
| Style diff / layer order | document deterministic reorder and missing `beforeLayerId` diagnostics | command fixtures define expected patch output |
| PMTiles/vector tile source | define source schema, resource policy, adapter transformer expectations | schema and resource-policy tests exist |

## Out of Scope

- Complete MapLibre style-spec parity.
- Full expression runtime evaluation.
- WebGPU renderer.
- Complete Cesium replacement.
- GeoParquet/DuckDB execution.
- Stable 3D Tiles rendering.

## Migration Rules

- `MapSpec.version` remains `"0.1"` until a breaking schema change is required.
- v0.2 feature work must be represented through additive schema fields,
  capability reports, diagnostics, examples, and tests.
- Experimental 2.5D/3D work must stay behind `extensions` and capability gates.
- Any unsupported public capability must fail with structured diagnostics rather
  than being silently accepted.

## Release Gates

- `pnpm build:schema`
- `pnpm check`
- smoke snapshot for rendering-adjacent changes
- visual snapshot in release-capable environments when adapters or visual
  fixtures change
- resource policy coverage when URLs, tiles, workers, or examples change

## First Execution Batch

The first W21 batch closes the two P1 audit gaps and then lands the expression
v0.2 contract:

1. MCP diagnostic failure path.
2. Command schema unknown-field rejection.
3. Expression v0.2 validator and capability alignment.
