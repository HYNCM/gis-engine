# v0.2 Release

## Release Notes

### Highlights

- Added generic `vector` sources with URL-template and TileJSON-style URL forms.
- Extended MapLibre transformation for vector source metadata, including
  `source-layer`, `minzoom`, `maxzoom`, and `attribution`.
- Expanded the v0.2 expression subset with `case`, `match`, `zoom`,
  `to-number`, and `to-string`.
- Added MCP tool output schemas and strict capability report validation.
- Added vector tile example, schema fixture, snapshot smoke coverage, and a real
  MapLibre visual snapshot scenario backed by generated local MVT.
- Added beta MapLibre adapter mapping for gated `fill-extrusion-lite` specs,
  including schema fixture, example, adapter contract coverage, and snapshot
  smoke and visual coverage.
- Kept the SceneView3D unsupported boundary and added an
  `extensions.scene3d` fixture for future v1 RFC work.
- Drafted the SceneView3D v1 RFC for camera, source, layer, resource policy,
  snapshot, query, and command contracts.
- Added SceneView3D v1 preparation alpha evidence: schema, commands, resource
  load plan, mock snapshot/query, MCP context, and release visual waiver gate.
- Added SceneView3D adapter feasibility evidence and recommended a narrow
  Three.js + 3DTilesRendererJS spike before any stable 3D runtime exposure.
- Added the first isolated `@gis-engine/scene3d-three-adapter` spike package
  for deterministic load-plan/resource-policy evidence without real renderer
  dependencies.

### Upgrade Notes

- MCP clients should read both `inputSchema` and `outputSchema` from tool
  descriptors.
- `capabilities` passed to context or explain tools must match
  `CapabilityReportSchema`; arbitrary capability objects now fail validation.
- `fill-extrusion-lite` requires an explicit 2.5D experimental request and is
  still experimental even though MapLibre beta mapping is available.
- `scene3d` remains reserved and returns `CAPABILITY.UNSUPPORTED`; the current
  SceneView3D work is a contract scaffold and alpha gate, not stable runtime
  rendering.

## Gate Checklist

### Required Evidence

| Change Type | Required Gates | Evidence |
| --- | --- | --- |
| schema or public type | `pnpm build:schema`, `pnpm test:schema-sync`, `pnpm test:schema` | schema diff, fixture result |
| command semantics | `pnpm test:commands`, `pnpm test:patch`, `pnpm test:runtime` | replay, dry-run, rollback, audit trace coverage |
| MCP tool behavior | `pnpm test:ai` | tool input/output schema and diagnostic paths |
| adapter transform | `pnpm test:adapter`, `pnpm test:snapshot:smoke` | transformer result and snapshot smoke |
| resource URL/tile/worker | `pnpm test:schema`, `pnpm test:resources` | resource policy diagnostics |
| visual behavior | `pnpm test:snapshot:visual` | report or explicit waiver |
| SceneView3D release visual boundary | `pnpm test:release:scene3d` | renderer visual evidence or coordinator waiver with deterministic no-bypass evidence |
| release candidate | `pnpm test:release:strict` | release runner output |

### SceneView3D W23 Promotion Readiness Gate

Stable runtime claims for SceneView3D remain blocked until the W23 promotion
readiness package is complete and the final quality gate has issued a
go/no-go decision.

Required evidence for any stable runtime claim:

- Promotion rubric that names the evidence matrix, owners, blockers, and thresholds.
- Browser matrix evidence from the release-capable browser runner with frame, console, and renderer diagnostics.
- Adapter promotion summary that consolidates load-plan, resource, runtime, snapshot, query, and release evidence.
- Guardrail diagnostics with explicit stable-runtime blocker codes.
- MCP decision recording whether promotion evidence summaries stay out of the public context.
- Docs alignment that reflects the same decision in public release/checklist documents.
- Quality gate verdict that explicitly says go/no-go before any stable `view.mode: "scene3d"` claim.

### Visual Snapshot Waiver

Waiver is allowed only when `code-reviewer` or `coordinator` labels the change
as non-rendering. Docs-only, schema-only, MCP-only, and command-only changes can
be waived if they do not touch renderer adapters, style transforms, snapshots,
resources, examples, or visual fixtures.

Release candidates must either run strict visual snapshots in a release-capable
environment or include a coordinator-approved waiver with follow-up evidence.

### Blocking Conditions

- Public schema change without schema-sync coverage.
- New command behavior without replay/dry-run/rollback coverage.
- Tool failure path that returns natural-language-only errors.
- URL/tile/worker change without resource policy validation.
- Rendering-adjacent change without smoke snapshot coverage.
- Release candidate without strict visual snapshot or explicit waiver.
- Stable SceneView3D claim without the W23 promotion package and quality gate verdict.
