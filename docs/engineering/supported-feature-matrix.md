# Supported Feature Matrix

This matrix defines the current MapLibre MVP surface after the 2026-05-17
v0.2 checkpoint and the 2026-05-18 SceneView3D v1 preparation pass. Features
outside this table must return structured diagnostics instead of being silently
accepted.

| Area | Supported surface | Notes |
| --- | --- | --- |
| Sources | `geojson`, `raster`, `pmtiles`, `vector` | PMTiles is transformed to a MapLibre vector source URL and has an IO-free runtime load-plan preflight through `createPMTilesRuntimeLoadPlan()` for URL policy, `metadata["source-layer"]`, range-policy requirements, and optional archive metadata budgets. `createPMTilesQueryEvidence()` can record point/bbox fixture evidence from caller-supplied decoded features, including source-layer, result caps, empty results, and blocked diagnostics, but does not fetch resources, parse archives, start workers, or return feature payloads. PMTiles runtime archive parsing and runtime feature query remain follow-up work. Generic vector tile URL templates support `tiles[]` and `url`. GeoParquet, FlatGeobuf, and GeoTIFF are public MapSpec source contracts with runtime blocked; GeoZarr remains blocked until schema, resource policy, diagnostics, and tests exist. |
| Layers | `background`, `raster`, `fill`, `line`, `circle`, `symbol-lite`, `fill-extrusion-lite`; stable layer `filter`, `minzoom`, `maxzoom` | `symbol-lite` maps to MapLibre `symbol`; `fill-extrusion-lite` maps to MapLibre `fill-extrusion` only when explicitly gated as experimental 2.5D. Layer filters must evaluate to boolean values, and layer zoom ranges stay within `0 <= minzoom <= maxzoom <= 24`. |
| Expressions | `get`, `has`, `all`, `any`, `!`, `==`, `!=`, `>`, `<`, `>=`, `<=`, `in`, `step`, `interpolate`, `literal`, `case`, `match`, `zoom`, `to-number`, `to-string` | `interpolate` supports linear interpolation only. Boolean/comparison operators are the supported filter subset for stable 2D layer filtering. |
| View | `center`, `zoom`, `bearing`, `pitch`, `bounds` | Bounds fitting remains command-level state in v0.1 and is consumed by Studio through `fitBounds`. |
| Commands | `addSource`, `removeSource`, `addLayer`, `removeLayer`, `setPaint`, `setLayout`, `setFilter`, `setLayerZoomRange`, `reorderLayer`, `setView`, `fitBounds`; SceneView3D prep commands `setSceneCamera`, `addSceneSource`, `removeSceneSource`, `addSceneLayer`, `removeSceneLayer`, `setSceneLayerVisibility` | `baseRevision` conflicts are rejected; `collectTrace` returns deterministic audit traces; v0.1 does not automatically retry or perform three-way merge. `setLayout.visibility`, `setFilter`, and `setLayerZoomRange` provide the stable AI-facing layer visibility contract, while `reorderLayer` remains the stable ordering primitive. SceneView3D prep commands mutate only `extensions.scene3d`. |
| Current MCP tools | `validate_spec`, `apply_commands`, `export_spec`, `get_context_summary`, `snapshot_spec`, `explain_spec`, `export_example_app` | These are the only tool names exposed by the current MCP server. CamelCase aliases are intentionally unsupported. Tool descriptors include input and output schemas. `get_context_summary` and `explain_spec` include extension-only SceneView3D context when present. |
| Natural-language generation evidence | `MapGenerationPromptPlannerInputSchema`, `MapGenerationPromptPlanSchema`, `MapGenerationRequestSchema`, `planMapGenerationRequest()`, `createMapGenerationCommandSkeleton()`, `createGenerationEvidenceBundle()` | Prompt-level handoff evidence composes existing tool contracts, planner confidence/provenance, command replay, spatial point/bbox query evidence, PMTiles fixture query summaries, diagnostics, snapshot/export readiness, and example manifests. Spatial query evidence records an explicit `capabilityGate` and is ready only when adapter query capability is declared or an auditable waiver is present; invalid geometry, missing/hidden layers, missing sources, unsupported sources, and empty results use stable `/spatialQueries/cases` diagnostics; result caps and deterministic fixture hashes are recorded without exposing feature payloads. Generated-app delivery exposes `spatialQueryReadiness` and source `queryEvidence` summaries so ready, follow-up-required, and blocked query states are schema-testable without parsing prose. `export_example_app` can carry a compact `generationEvidence` manifest summary while remaining side-effect free. Planner inputs use prompt hashes plus structured intent and reject raw prompt retention by default. It does not expose a `generate_map_app` or `spatial_query` MCP tool. Spatial-analysis stays readiness-only, and scene browsing stays under `extensions.scene3d`. |
| Snapshot smoke | data-url adapter smoke plus SceneView3D release visual gate contract | Required deterministic gate. Must run under Node/Vitest and must not require real browser canvas, GPU, or WebGL. SceneView3D release mode requires renderer visual evidence or a coordinator waiver, and waivers cannot bypass pending resources or snapshot/query diagnostics. |
| Snapshot visual | Playwright + real browser MapLibre GL canvas | Conditional/release validation for pixel health and optional baseline diff. Current scenarios cover GeoJSON, generated local MVT, and gated `fill-extrusion-lite`. This is separate from `pnpm check` unless a release job explicitly requires visual snapshots. |
| MapLibre adapter binding | Transformer + adapter contract MVP | Visual snapshot tests exercise real browser MapLibre GL canvas health. |
| Experimental boundary | `fill-extrusion-lite`, `extensions.scene3d`, `@gis-engine/scene3d` scaffold, `@gis-engine/scene3d-three-adapter` spike | `fill-extrusion-lite` is beta-supported by the MapLibre adapter behind explicit gates. `extensions.scene3d` has a formal schema, source URL policy checks, layer-source validation, command patches, loader resource load plan gate, mock snapshot/query contracts, MCP context summaries, and release visual waiver rules. The Three.js adapter spike currently creates deterministic load-plan/resource-policy evidence only; `view.mode: "scene3d"` still returns unsupported diagnostics. |

Explicitly out of scope for the current stable surface: full symbol placement,
terrain rendering, globe, production 3D Tiles loading, glTF rendering, custom
WebGL layers, heatmap, clusters, SceneView3D runtime, and full Mapbox expression
compatibility.
