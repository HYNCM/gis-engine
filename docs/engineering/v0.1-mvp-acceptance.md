# v0.1 MVP 验收标准

## 目标

v0.1 的目标不是做完整地图引擎，而是证明 GIS Engine 的 AI 原生闭环可用、可测、可复现。

v0.1 必须完成：

```txt
MapSpec -> validate -> render -> command modify -> snapshot -> export
```

任何不能服务这个闭环的能力，都进入 backlog 或 experimental。

## 范围边界

### 必须交付

| 类别 | v0.1 范围 |
| --- | --- |
| 包 | `@gis-engine/engine`、`@gis-engine/ai` |
| 渲染 | MapLibre adapter MVP 作为默认后端，当前覆盖 `MapSpec -> MapLibre style` transformer、adapter contract 和 data-url snapshot smoke |
| 实验渲染 | `webgl2-lite` 不属于 v0.1 阻断范围 |
| 数据源 | GeoJSON、raster tiles、PMTiles |
| 图层 | background、raster、fill、line、circle、symbol-lite |
| 2.5D | `fill-extrusion-lite` experimental，不阻塞发布 |
| API | `createMap`、`apply`、`exportSpec`、`validate`、`queryFeatures`、`snapshot`、`destroy` |
| AI/MCP tools | 当前交付 `validate_spec`、`apply_commands`、`export_spec`、`get_context_summary`；计划补充 `snapshot_spec`、`explain_spec`、`export_example_app` |
| 文档 | README、MapSpec、commands、diagnostics、adapter、snapshot、MCP tools |
| 示例 | 当前交付 basic GeoJSON、AI map edit；raster、PMTiles 进入后续示例补齐 |
| 测试 | schema、commands、adapter contract、snapshot、AI tool contract、resource release、perf smoke |

### 明确不做

- 完整 3D Tiles。
- terrain。
- glTF object layer。
- SceneView3D 稳定版。
- WebGPU。
- GeoParquet 和 DuckDB WASM。
- 完整 Mapbox expression。
- 完整 symbol collision、glyph shaping、复杂 label placement。
- 完整 OGC 服务矩阵。
- 企业 GIS 编辑平台。

### Experimental 进入条件

实验能力必须满足：

- 只能通过 `capabilities.experimental` 启用。
- 必须写入 `extensions`，不能新增 v0.1 核心字段。
- 必须有诊断 code。
- 必须有至少 1 个示例和 1 个 contract test。
- 文档必须标注风险和替代路径。

## 包边界

### `@gis-engine/engine`

负责：

- `MapSpec` schema 和 TypeScript 类型。
- spec normalize、validate、diff、patch。
- command dry-run、apply、rollback。
- diagnostics。
- renderer adapter 接口。
- MapLibre adapter。
- snapshot、query、destroy。

禁止：

- 依赖 `@gis-engine/ai`。
- 依赖 MCP server runtime。
- 暴露 renderer 私有对象给 AI tools。

### `@gis-engine/ai`

负责：

- MCP tools。
- AI-friendly tool input/output schemas。
- 当前 snake_case tools：`validate_spec`、`apply_commands`、`export_spec`、`get_context_summary`。
- 后续 snake_case tools：`snapshot_spec`、`explain_spec`、`export_example_app`。
- AI eval fixtures。

依赖方向：

```txt
@gis-engine/ai -> @gis-engine/engine
@gis-engine/engine -X-> @gis-engine/ai
```

## v0.1 Definition of Done

v0.1 只有在以下条件全部满足时才算完成：

| Gate | 验收标准 |
| --- | --- |
| Schema | `MapSpec v0.1`、commands、tool input/output 都有 schema 和类型 |
| Schema 同步 | CI 能检测 schema 和类型不同步 |
| Commands | `addSource`、`addLayer`、`setPaint`、`setLayout`、`setView`、`fitBounds`、`removeLayer`、`reorderLayer` 可 dry-run 和 replay |
| Diagnostics | 所有失败返回结构化 code、path 和修复建议 |
| Adapter | MapLibre adapter MVP 通过 renderer contract tests；真实浏览器 MapLibre GL canvas binding 后置到未来 `RFC-QC-*` |
| Snapshot smoke | Node/Vitest smoke 必跑且阻断，可验证 adapter `load -> snapshot -> exportSpec` 的结构化结果，不依赖真实 GPU、WebGL 或浏览器 canvas |
| Snapshot visual | Playwright + 真实浏览器 + 真实 MapLibre GL canvas 为条件/发布验收项；用于像素健康检查和可选 baseline diff |
| Examples | basic GeoJSON 和 AI map edit 示例可作为 CI fixtures；raster、PMTiles 示例不阻塞 Wave 1 |
| AI tools | MCP tool contract tests 通过；只接受 snake_case tool names |
| Performance | perf smoke tests 不超过 v0.1 budget |
| Resource release | `destroy()` 后 listener、worker、raf、WebGL resources 释放 |
| Docs | README 第一屏说明定位、快速示例、当前不做什么 |

v0.1 还必须满足 [implementation-playbook.md](./implementation-playbook.md) 中定义的默认策略：schema-sync、冲突拒绝策略、snapshot baseline 管理、URL policy、SuggestedFix governance 和 resource release assertions。

MCP tool 命名是 v0.1 public contract 的一部分。`validateMap`、`applyCommands`、`snapshotMap`、`explainMap`、`exportSpec`、`exportExampleApp` 这类 camelCase 名称不作为 MCP aliases 暴露；如果需要在 TypeScript 内部保留 helper，可与 MCP tool name 分开记录。

## 示例和 fixtures

v0.1 Wave 1 必须包含以下 fixtures：

```txt
examples/
  basic-geojson/
    map.json
    data/points.geojson
  ai-map-edit/
    before.map.json
    commands.json
```

每个示例必须包含：

- 自然语言需求。
- `MapSpec`。
- TypeScript 使用方式。
- command 修改流程。
- snapshot 预期。
- 失败诊断样例。

`raster-basemap` 和 `pmtiles-local` 示例应在后续补齐，但不作为 Wave 1 docs/release gate。

## 发布策略

- v0.1 使用单一 workspace，公开两个包。
- `engine` 和 `ai` 版本保持一致。
- npm 发布前必须通过全部 required CI gates。
- `pnpm check` 只能依赖确定性的 build、schema、Vitest/Node smoke 和 contract gates，不得依赖真实 GPU/WebGL。
- v0.1 不做自动 retry，不做 three-way merge；revision conflict 由调用方显式刷新、rebase 和重试。
- `MapSpec.version` 与 npm package version 独立，`MapSpec` 使用语义迁移策略。
- 任何 breaking schema change 必须附带 migration。

## Backlog

以下能力进入 backlog，不得阻塞 v0.1：

- Generic vector tile URL template。
- WMS/WMTS。
- FlatGeobuf。
- Heatmap、cluster、flow line、large point cloud。
- SceneView3D beta。
- 3D Tiles adapter。
- WebGPU renderer。
- GeoParquet analytics。
- visual editor 和 devtools UI。
