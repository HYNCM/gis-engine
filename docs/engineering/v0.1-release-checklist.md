# v0.1 Release Checklist

This checklist captures the v0.1 RC hardening gate for the current implementation. It tracks documentation, scripts, schema contracts, and release-readiness evidence.

## Contract Surface

- [x] Public runtime API is described as `createMap`, `MapRuntime.apply`, `validate`, `snapshot`, `queryFeatures`, `exportSpec`, and `destroy`.
- [x] Current MCP tools are documented exactly as `validate_spec`, `apply_commands`, `export_spec`, `get_context_summary`, `snapshot_spec`, `explain_spec`, and `export_example_app`.
- [x] No camelCase MCP aliases are documented or implied.
- [x] `MapSpec.version` remains independent from npm package version.

## Snapshot Gates

- [x] Snapshot smoke is documented as the deterministic required gate.
- [x] Snapshot smoke covers adapter contract behavior such as `load -> snapshot -> exportSpec`, structured diagnostics, and data-url snapshot output.
- [x] Snapshot smoke and `pnpm check` do not depend on a real browser canvas, GPU, or WebGL.
- [x] Snapshot visual is documented as a separate Playwright/browser gate for real MapLibre GL canvas pixel health and optional baseline diff.
- [x] Visual snapshot failures are release-blocking only when the release job or `GIS_ENGINE_REQUIRE_VISUAL_SNAPSHOT=1` explicitly requires them.

## Command And Concurrency Policy

- [x] `baseRevision` conflict behavior is documented as reject-and-report.
- [x] v0.1 does not perform automatic retry.
- [x] v0.1 does not perform three-way merge.
- [x] Callers are responsible for refreshing the latest spec, rebasing intended commands, and retrying explicitly.
- [x] Single-runtime serialization is documented as a local ordering guard, not a durable command queue.

## Adapter Scope

- [x] `MapLibreAdapter` is described as the v0.1 MVP transformer/adapter contract binding.
- [x] The current adapter supports deterministic data-url smoke snapshots.
- [x] Real browser MapLibre GL canvas binding is deferred to a future `RFC-QC-*` contract-quality-control change.
- [x] Unsupported feature paths return structured diagnostics rather than being silently accepted.

## Release Evidence

- [x] `pnpm build:schema` passes when schema files change.
- [x] `pnpm check` passes with deterministic build/test gates.
- [x] `pnpm check` remains free of real GPU/WebGL requirements.
- [x] README, MVP acceptance, supported feature matrix, and this checklist agree on tool names and snapshot gate semantics.
- [x] Any later expansion of public MCP tools includes schema-sync coverage, MCP contract tests, and a release-note entry.

## Remaining Before Publish

- [x] Strict visual snapshot gate passes in a release-capable environment.
- [ ] Release notes are written.
- [ ] npm publish dry-run and package file review are complete.
