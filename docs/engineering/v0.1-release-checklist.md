# v0.1 Release Checklist

This checklist captures the Wave 1 docs/release gate for the current implementation. It is intentionally scoped to release readiness notes and must not require code, test, or `package.json` changes.

## Contract Surface

- [ ] Public runtime API is described as `createMap`, `MapRuntime.apply`, `validate`, `snapshot`, `queryFeatures`, `exportSpec`, and `destroy`.
- [ ] Current MCP tools are documented exactly as `validate_spec`, `apply_commands`, `export_spec`, and `get_context_summary`.
- [ ] Future MCP tools are documented as `snapshot_spec`, `explain_spec`, and `export_example_app`.
- [ ] No camelCase MCP aliases are documented or implied.
- [ ] `MapSpec.version` remains independent from npm package version.

## Snapshot Gates

- [ ] Snapshot smoke is documented as the deterministic required gate.
- [ ] Snapshot smoke covers adapter contract behavior such as `load -> snapshot -> exportSpec`, structured diagnostics, and data-url snapshot output.
- [ ] Snapshot smoke and `pnpm check` do not depend on a real browser canvas, GPU, or WebGL.
- [ ] Snapshot visual is documented as a separate Playwright/browser gate for real MapLibre GL canvas pixel health and optional baseline diff.
- [ ] Visual snapshot failures are release-blocking only when the release job or `GIS_ENGINE_REQUIRE_VISUAL_SNAPSHOT=1` explicitly requires them.

## Command And Concurrency Policy

- [ ] `baseRevision` conflict behavior is documented as reject-and-report.
- [ ] v0.1 does not perform automatic retry.
- [ ] v0.1 does not perform three-way merge.
- [ ] Callers are responsible for refreshing the latest spec, rebasing intended commands, and retrying explicitly.
- [ ] Single-runtime serialization is documented as a local ordering guard, not a durable command queue.

## Adapter Scope

- [ ] `MapLibreAdapter` is described as the v0.1 MVP transformer/adapter contract binding.
- [ ] The current adapter supports deterministic data-url smoke snapshots.
- [ ] Real browser MapLibre GL canvas binding is deferred to a future `RFC-QC-*` contract-quality-control change.
- [ ] Unsupported feature paths return structured diagnostics rather than being silently accepted.

## Release Evidence

- [ ] `pnpm build:schema` passes when schema files change.
- [ ] `pnpm check` passes with deterministic build/test gates.
- [ ] `pnpm check` remains free of real GPU/WebGL requirements.
- [ ] README, MVP acceptance, supported feature matrix, and this checklist agree on tool names and snapshot gate semantics.
- [ ] Any later expansion of public MCP tools includes schema-sync coverage, MCP contract tests, and a release-note entry.
