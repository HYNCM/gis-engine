# Migrating from v0.2 to v0.3

## Breaking Changes

### 1. `hashPrompt()` format change
- **Before (v0.2)**: `hashPrompt("test")` returned a 16-character hex string like `"a1b2c3d4e5f67890"`
- **After (v0.3)**: Returns `sha256:<32-hex>` format like `"sha256:a1b2c3d4e5f67890a1b2c3d4e5f67890"`
- **Why**: The `normalizeWorkbenchProviderPlan()` function validates promptHash against `/^sha256:[A-Za-z0-9._:-]{1,96}$/`. The old format was rejected.
- **Action**: If you store or compare prompt hashes, update your expectations. Hash values are now 39 characters long (7 prefix + 32 hex).

### 2. Generate pipeline behavior change
- **Before**: Generate pipeline used hardcoded `intent: { targetDomains: ["feature-display"] }`
- **After**: Pipeline resolves intent from provider (HTTP call for real providers, deterministic for mock)
- **Action**: If you relied on the hardcoded feature-display domain, mock provider still produces the same output. Real providers now produce AI-resolved intent.

## New Features

### Provider HTTP layer
- Real OpenAI-compatible API calls from the CLI generate pipeline
- `--api-key` flag and `DEEPSEEK_API_KEY` / `OPENAI_API_KEY` env var support
- `--timeout` flag for provider request timeout (default: 20000ms)

### New exports
- `callProvider()` -- HTTP client for OpenAI-compatible providers
- `resolveProviderProfile()` -- build a provider profile from config
- `readProviderApiKey()` -- read API key from environment
- `CLI_API_KEY_ENVS` -- maps provider IDs to env var names
- `DEFAULT_PROVIDER_TIMEOUT_MS`, `DEFAULT_PROVIDER_BYTE_CAP` -- constants
- Types: `ProviderProfile`, `ProviderCallInput`, `ProviderCallResult`, `ProviderOutput`, `ProviderConfidence`, `ProviderCallDiagnostic`

### CLI flags
- `--api-key <key>` -- API key for provider (overrides env var)
- `--timeout <ms>` -- Provider request timeout in milliseconds

## Upgrade Steps

1. Update `package.json`: `"@gis-engine/cli": "^0.3.0"`
2. Run `npm install` or `pnpm install`
3. If you use `hashPrompt()` programmatically, update tests to expect `sha256:<hex>` format
4. If you use real providers, set your API key env var or pass `--api-key`
5. No changes needed for mock provider usage
