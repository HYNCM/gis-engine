# Migrating from v1.4 to v1.5

This guide covers the upgrade from GIS Engine `v1.4.x` to `v1.5.0`.

## Who Should Read This

- SDK consumers using `@gis-engine/engine`
- MCP / AI consumers using `@gis-engine/ai`
- CLI users using `create-gis-map`

## Upgrade Summary

`v1.5.0` focuses on SDK product maturity: expanded test coverage, CI matrix
improvements, performance benchmarks, and documentation quality. No breaking
changes to public APIs.

## Package Line

Update to:

```json
{
  "dependencies": {
    "@gis-engine/engine": "^1.5.0",
    "@gis-engine/ai": "^1.5.0",
    "@gis-engine/cli": "^1.5.0"
  }
}
```

`@gis-engine/scene3d` and `@gis-engine/scene3d-three-adapter` remain
experimental packages. Stable `view.mode: "scene3d"` runtime is not promoted
in `v1.5.0`.

## What Changed in v1.5

### CLI test coverage expansion

The CLI test suite expanded from 141 to 215+ test cases, eliminating zero-
coverage modules:

- `lintMapSpec()` and `formatLintText()` now have dedicated unit tests
  covering valid specs, file-not-found, malformed JSON, non-object JSON,
  diagnostic counting, and text formatting.
- Community template registry (`registerCommunityTemplate`,
  `unregisterCommunityTemplate`, `getCommunityTemplate`,
  `listCommunityTemplates`, `isCommunityTemplateName`,
  `parseCommunityTemplateName`, `validateCommunityManifest`,
  `generateCommunityTemplate`) now has full test coverage.
- `generate()` pipeline tests expanded to cover dryRun mode, result structure,
  default prompt, evidence structure, and provider error handling.
- `main()` dispatch tests added for `--help`, `--version`, error paths
  (missing project name, unknown template, existing directory), and scaffold
  smoke tests.

### CI multi-version matrix

The CI workflow now tests across:

- **Node 22** (minimum supported, all platforms)
- **Node 24** (ubuntu-latest lint + build-test)
- **macOS** (lint + build-test with Node 22)

If your environment uses a different Node version, ensure it is `>= 22.13.0`
as specified in `engines`.

### Performance benchmarks

New `applyCommands` batch performance benchmarks added:

- 100 commands: < 500ms budget
- 500 commands: < 2000ms budget
- 1000 commands: < 5000ms budget
- Mixed command types (setView + setPaint, 400 commands): < 3000ms budget

Actual measured performance is significantly under budget (1000 commands
complete in ~16ms on reference hardware).

### Documentation

- Core-vs-Extension matrix now documented in architecture docs
- Docs boundary regression tests hardened
- Feature matrix and CI test strategy docs synchronized

## Behavior And Contract Notes

### MCP surface

No changes. The public MCP surface remains these 14 MCP tools:
`validate_spec`, `apply_commands`, `export_spec`, `get_context_summary`,
`snapshot_spec`, `explain_spec`, `export_example_app`, `diff_specs`,
`generate_spec`, `inspect_data`, `edit_spec`, `query_features`,
`style_recommend`, `transform_data`.

### CLI surface

- `--lint` flag: validates and lints a MapSpec JSON file (lighter than
  `--preflight`). Available since v1.4.0.

```bash
npm exec --package @gis-engine/cli@latest -- create-gis-map --lint ./map.json
npm exec --package @gis-engine/cli@latest -- create-gis-map --lint ./map.json --json
```

- Community templates: `community:geojson-explorer` and `community:pmtiles-viewer`
  are available as built-in community templates.

```bash
npm exec --package @gis-engine/cli@latest -- create-gis-map my-map -t community:geojson-explorer
```

### Expression engine (since v1.4.0)

If upgrading from before v1.4.0, note the expression engine additions:

- Arithmetic operators: `+`, `-`, `*`, `/`
- `coalesce`, exponential/cubic-bezier `interpolate`
- String expressions: `concat`, `upcase`, `downcase`
- `heatmap-density` expression

### Layer types (since v1.4.0)

- `heatmap` layer type
- `symbol` layer type (full version; `symbol-lite` retained for backward
  compatibility)

## Recommended Upgrade Steps

1. Bump package versions to `^1.5.0`.
2. Re-run your existing validation / snapshot / review pipeline:
   ```bash
   pnpm build:schema
   pnpm check
   ```
3. If using CI, verify your pipeline works with Node 22+ (Node 20 is not
   supported).
4. Run `create-gis-map --lint ./map.json` on existing MapSpec files to
   validate against the current schema.
