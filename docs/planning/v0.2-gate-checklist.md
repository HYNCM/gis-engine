## SceneView3D W23 Promotion Readiness Gate

Stable runtime claims for SceneView3D remain blocked until the W23 promotion
readiness package is complete and the final quality gate has issued a
go/no-go decision.

Required evidence for any stable runtime claim:

- Promotion rubric that names the evidence matrix, owners, blockers, and
  thresholds.
- Browser matrix evidence from the release-capable browser runner with frame,
  console, and renderer diagnostics.
- Adapter promotion summary that consolidates load-plan, resource, runtime,
  snapshot, query, and release evidence.
- Guardrail diagnostics with explicit stable-runtime blocker codes.
- MCP decision recording whether promotion evidence summaries stay out of the
  public context.
- Docs alignment that reflects the same decision in public release/checklist
  documents.
- Quality gate verdict that explicitly says go/no-go before any stable
  `view.mode: "scene3d"` claim.

## Visual Snapshot Waiver

Waiver is allowed only when `code-reviewer` or `coordinator` labels the change
as non-rendering. Docs-only, schema-only, MCP-only, and command-only changes can
be waived if they do not touch renderer adapters, style transforms, snapshots,
resources, examples, or visual fixtures.

Release candidates must either run strict visual snapshots in a release-capable
environment or include a coordinator-approved waiver with follow-up evidence.

## Blocking Conditions

- Public schema change without schema-sync coverage.
- New command behavior without replay/dry-run/rollback coverage.
- Tool failure path that returns natural-language-only errors.
- URL/tile/worker change without resource policy validation.
- Rendering-adjacent change without smoke snapshot coverage.
- Release candidate without strict visual snapshot or explicit waiver.
- Stable SceneView3D claim without the W23 promotion package and quality gate
  verdict.
