# v0.2 Release Note Draft

## Highlights

- Added generic `vector` sources with URL-template and TileJSON-style URL forms.
- Extended MapLibre transformation for vector source metadata, including
  `source-layer`, `minzoom`, `maxzoom`, and `attribution`.
- Expanded the v0.2 expression subset with `case`, `match`, `zoom`,
  `to-number`, and `to-string`.
- Added MCP tool output schemas and strict capability report validation.
- Added vector tile example, schema fixture, snapshot smoke coverage, and a real
  MapLibre visual snapshot scenario backed by generated local MVT.
- Defined the `fill-extrusion-lite` experimental 2.5D gate and the SceneView3D
  unsupported boundary.

## Upgrade Notes

- MCP clients should read both `inputSchema` and `outputSchema` from tool
  descriptors.
- `capabilities` passed to context or explain tools must match
  `CapabilityReportSchema`; arbitrary capability objects now fail validation.
- `fill-extrusion-lite` requires an explicit 2.5D experimental request and is
  not yet rendered by the MapLibre MVP adapter.
- `scene3d` remains reserved and returns `CAPABILITY.UNSUPPORTED`.

## Validation

- `pnpm build:schema` passed.
- `pnpm check` passed.
- `pnpm test:snapshot:visual` passed with 2 Playwright tests.
- `GIS_ENGINE_REQUIRE_VISUAL_SNAPSHOT=1 pnpm test:snapshot:visual` passed
  outside the macOS sandbox; release runners need Chromium/WebGL permissions.
