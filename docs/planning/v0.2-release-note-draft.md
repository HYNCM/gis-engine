# v0.2 Release Note Draft

## Highlights

- Added generic `vector` sources with URL-template and TileJSON-style URL forms.
- Extended MapLibre transformation for vector source metadata, including
  `source-layer`, `minzoom`, `maxzoom`, and `attribution`.
- Expanded the v0.2 expression subset with `case`, `match`, `zoom`,
  `to-number`, and `to-string`.
- Added MCP tool output schemas and strict capability report validation.
- Added vector tile example, schema fixture, snapshot smoke coverage, and a real
  MapLibre visual snapshot scenario backed by generated local MVT.
- Added beta MapLibre adapter mapping for gated `fill-extrusion-lite` specs,
  including schema fixture, example, adapter contract coverage, and snapshot
  smoke and visual coverage.
- Kept the SceneView3D unsupported boundary and added an
  `extensions.scene3d` fixture for future v1 RFC work.
- Drafted the SceneView3D v1 RFC for camera, source, layer, resource policy,
  snapshot, query, and command contracts.
- Added SceneView3D v1 preparation alpha evidence: schema, commands, resource
  load plan, mock snapshot/query, MCP context, and release visual waiver gate.
- Added SceneView3D adapter feasibility evidence and recommended a narrow
  Three.js + 3DTilesRendererJS spike before any stable 3D runtime exposure.

## Upgrade Notes

- MCP clients should read both `inputSchema` and `outputSchema` from tool
  descriptors.
- `capabilities` passed to context or explain tools must match
  `CapabilityReportSchema`; arbitrary capability objects now fail validation.
- `fill-extrusion-lite` requires an explicit 2.5D experimental request and is
  still experimental even though MapLibre beta mapping is available.
- `scene3d` remains reserved and returns `CAPABILITY.UNSUPPORTED`; the current
  SceneView3D work is a contract scaffold and alpha gate, not stable runtime
  rendering.

- `pnpm check` passed.
