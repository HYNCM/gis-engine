# MLD-004 MapLibre Drift Go-No-go Gate

## Decision

**No-go for MapLibre package movement in this batch.**

The MLD audit is closed as compatibility and planning evidence, not as a
dependency upgrade approval. The repository should stay on the current
`maplibre-gl` package range. A future package movement task must refresh
official package/changelog evidence in that run, update any example/runtime
bundle handling needed by the target version, and produce accepted visual
snapshot evidence before it can become a go-candidate.

This decision is blocking only for MapLibre dependency movement. It does not
block the current docs/planning closure.

## Future Go-Candidate Requirements

A future MapLibre package movement task may become a go-candidate only after it
records all of the following:

1. Official MapLibre release or npm evidence checked during that run.
2. Exact package and lockfile diff.
3. Adapter, resource, AI, docs, smoke snapshot, and full `pnpm -s check`
   outputs from the same dependency state.
4. Browser visual evidence for GeoJSON, generated local MVT, and gated
   `fill-extrusion-lite` scenarios.
5. Strict visual evidence for release-capable claims, or an explicit
   coordinator waiver that keeps package movement conditional.
6. Example loading compatibility for the candidate runtime bundle shape.

## Non-Goals Reaffirmed

- No package upgrade in this MLD closure.
- No PMTiles archive parsing or vector tile decoding.
- No hidden network fetches, workers, or external services.
- No MCP alias.
- No stable SceneView3D runtime promotion.
